## Вопрос 1: Сколько байт занимает тип int на вашей системе? Совпадает ли это с разницей между адресами элементов массива из предыдущего задания?

# Ответ: int занимает 4 байта. Да, совпадает с разницей между &a[0] и &a[1], потому что элементы массива лежат подряд.

## Вопрос 2: Почему размер указателя (8 байт) совпадает с размером long на 64-битной системе?

# Ответ: На 64-битной системе адрес памяти занимает 8 байт (64 бита). long тоже часто занимает 8 байт, поэтому размеры совпадают.

## Вопрос 3: Сколько байт занимает char? Почему именно столько?

# Ответ: char занимает 1 байт. Это минимальная адресуемая единица памяти, которой хватает для хранения одного символа.

## Вопрос 4: Почему float (4 байта) и int (4 байта) занимают одинаковый размер, но хранят разные числа?

# Ответ: Одинаковый размер (4 байта), но разный способ записи:

# int — просто двоичное число

# float — по стандарту IEEE 754 (знак + порядок + мантисса)

 ###  Часть 2 
  ##  Ответы на вопросы
## 1. Принцип фон Неймана (команды и данные в одной памяти) — значение для C-программы?

## Ответ:
# Это позволяет программе менять свой код во время выполнения (самомодифицирующийся код). Программа может создавать новые инструкции в памяти и выполнять их.

# Риски: Вирусы могут записать вредоносный код в память и заставить процессор его выполнить (переполнение буфера).

## 2. Как разные области памяти иллюстрируют принцип фон Неймана?

## Ответ:
# Хотя переменные лежат в разных местах (глобальные, стек, куча), у всех у них есть адреса в одном общем пространстве. Процессор не различает, где данные, а где код — он просто читает байты по адресам. Это и есть единое линейное адресное пространство.

## 3. Что такое "слово" процессора и как его размер влияет на...

## Ответ:
# Слово — это количество бит, которое процессор обрабатывает за одну операцию (32 или 64 бита).

# Что влияет	32-битный процессор	64-битный процессор
# Макс. память	4 ГБ (2³² байт)	16 млн ТБ (2⁶⁴ байт)
# Размер int	обычно 4 байта	обычно 4 байта
# Размер long	4 байта	8 байт
# Размер указателя	4 байта	8 байт
## 4. Процессор читает 01000001 — как поймёт, что это?

## Ответ:
# Процессор сам не знает — всё зависит от инструкции, которая этот байт запросила:

# Если инструкция add (сложение) — это число 65

# Если инструкция работы с символами — это буква 'A' (ASCII код 65)

# Какой компонент решает? УУ (Устройство Управления) — оно декодирует инструкцию и говорит остальным, что делать с данными.

## 5. Что такое "канарейка стека" (stack canary)?

## Ответ:
# Это защита от переполнения буфера.

# Как работает:

# При входе в функцию кладётся секретное значение (канарейка) между локальными переменными и адресом возврата

# Перед выходом из функции проверяется, не изменилась ли канарейка

# Если изменилась — значит, кто-то пытался перезаписать стек, программа аварийно завершается

# Расположение на стеке:

# text
# [локальные переменные] [КАНАРЕЙКА] [адрес возврата] [параметры]
## 6. Что такое бит NX? Чем отличается от канарейки?

## Ответ:
# NX-бит (No eXecute) — защита на уровне страниц памяти. Помечает области памяти как "только данные" — оттуда нельзя выполнять код.

# Канарейка	NX-бит
# Защищает конкретную функцию	Защищает целые страницы памяти
# Проверяет целостность стека	Запрещает выполнение кода из стека/кучи
# Работает во время выполнения	Работает на уровне процессора/ОС
# Дополняют друг друга: канарейка ловит попытки перезаписи, NX не даёт выполнить код, даже если его записали.
